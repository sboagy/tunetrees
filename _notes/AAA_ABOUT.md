# About `_notes/`

Development notes and working documents. **Not guaranteed to be current or accurate.**

## What goes here
- Implementation plans and design notes
- Session handoff documents
- Debugging notes and investigations
- Historical context (how/why decisions were made)

## What goes in `docs/`
- User-facing documentation
- Developer setup guides
- Architecture documentation meant to be maintained

When in doubt: if it needs to stay accurate, put it in `docs/`. If it's reference material that might get stale, put it here.
